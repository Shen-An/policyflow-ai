# 03. UI/UX 设计合同

状态：Draft  
上游：`01-scope-and-boundaries.md`  
用于：所有页面和通用组件实现

## 本文负责

- 视觉令牌；
- 交互、文案和状态表达；
- 通用组件合同；
- 响应式和可访问性基线；
- 第三方组件安全。

## 本文不负责

- API 请求和缓存；
- 路由权限；
- 具体业务排期。

---

## 1. 设计方向

关键词：可信、克制、企业工具、证据优先、状态清晰。

禁止：大面积娱乐化渐变、拟人头像、无意义动效、仅靠颜色表达状态、为了“像聊天产品”牺牲引用信息。

## 2. 间距

使用 4px 网格：

| Token | 值 | 用途 |
|---|---:|---|
| `space-1` | 4px | 图标间距 |
| `space-2` | 8px | 紧凑组件 |
| `space-3` | 12px | 列表内部 |
| `space-4` | 16px | 默认间距 |
| `space-6` | 24px | 卡片和区块 |
| `space-8` | 32px | 页面区块 |
| `space-12` | 48px | 大型空状态 |

不新增 13px、18px 等无令牌魔法间距。

## 3. 字体

| 角色 | 字号 | 字重 | 行高 |
|---|---:|---:|---:|
| 页面标题 | 24px | 600 | 32px |
| 区块标题 | 18px | 600 | 28px |
| 卡片标题 | 16px | 600 | 24px |
| 正文 | 14px | 400 | 22px |
| 辅助文字 | 12px | 400 | 18px |
| 表头 | 12px | 600 | 18px |

使用系统字体栈，不在 MVP 引入额外 Web Font。

## 4. 颜色

| 角色 | 值 | 用途 |
|---|---|---|
| 背景 | `#F8FAFC` | 页面背景 |
| 表面 | `#FFFFFF` | 卡片、弹窗 |
| 主文字 | `#0F172A` | 标题、正文 |
| 次文字 | `#475569` | 说明 |
| 边框 | `#E2E8F0` | 分隔 |
| 主操作 | `#2563EB` | 主要 CTA、选中状态 |
| 成功 | `#15803D` | 健康、通过 |
| 警告 | `#B45309` | 待处理、低置信度 |
| 危险 | `#B91C1C` | 驳回、失败、删除 |

强调色不用于所有可点击元素。状态同时提供图标或文字。

## 5. 通用交互

- 异步按钮显示进行中并防重复；
- 破坏性操作必须二次确认；
- 筛选、分页优先同步 URL；
- 空状态必须说明原因和下一步；
- Toast 只承载短暂结果，关键错误保留在页面；
- 禁用状态必须说明原因；
- 长任务使用明确状态和有限轮询；
- 未发送聊天内容仅在当前标签页会话内保留；
- 页面离开前保护未保存草稿。

## 6. 文案合同

| 场景 | 文案 |
|---|---|
| 发送 | `发送问题` |
| 上传 | `上传文档` |
| 无知识库 | `还没有可访问的知识库，请联系知识库管理员授权。` |
| 无证据 | `当前知识库未找到可靠依据，建议联系相关部门确认。` |
| 无权限 | `你没有访问此功能的权限。若认为这是错误，请联系系统管理员。` |
| 会话过期 | `登录状态已过期，请重新登录。` |
| 驳回确认 | `确认驳回这条 FAQ？驳回原因将被记录。` |

错误文案需包含“发生了什么”和“用户下一步能做什么”。

## 7. 通用组件合同

| 组件 | 负责 | 不负责 |
|---|---|---|
| `PageHeader` | 标题、说明、主操作 | API 请求 |
| `DataTable` | 表格框架、加载、空状态 | 业务筛选逻辑 |
| `StatusBadge` | 状态语义 | 只用颜色表达 |
| `EmptyState` | 原因和下一步 | 含糊的“暂无数据” |
| `ErrorState` | 错误、request_id、重试 | 展示堆栈 |
| `ConfirmDialog` | 明确危险动作 | 含糊的“确定吗” |
| `AsyncButton` | 提交态和防重复 | 吞掉错误 |
| `PermissionGate` | 操作级体验门禁 | 替代后端鉴权 |
| `CitationCard` | 来源和片段 | 隐藏文档来源 |
| `IndexStatus` | 索引状态和重试 | 无限轮询 |

## 8. 响应式

- `<768px`：单列、侧栏抽屉、表格横向滚动或卡片化；
- `768～1279px`：可收起侧栏、详情抽屉；
- `>=1280px`：完整侧栏，问答与引用可双栏。

管理页面以桌面和平板为优先，移动端必须可查看和完成关键操作。

## 9. 可访问性

- 输入有可见 Label；
- 键盘可操作菜单、对话框和引用；
- 焦点清晰；
- 对话框正确管理焦点；
- 图标按钮有可访问名称；
- 错误与字段关联；
- 尊重 `prefers-reduced-motion`；
- 状态不只靠颜色。

## 10. Registry 安全

- 只从官方 shadcn Registry 拉取组件；
- 生成后审查 diff；
- 不复制来源不明的组件脚本或远程 CSS；
- 新依赖需说明用途、体积和替代性；
- 所有生成组件接受 lint、类型、测试和可访问性门禁。

## 11. 完成条件

- 六个维度均有合同：文案、视觉、颜色、字体、间距、Registry 安全；
- 页面状态可访问且含义明确；
- 未引入页面级随意样式；
- 未为业务特例破坏通用组件边界。
