# 09. 测试、安全与质量门禁

状态：Draft  
适用：每个 Phase 验收以及 F7 生产收口

## 本文负责

- 单元、组件、契约和 E2E 策略；
- 浏览器安全；
- 性能、可靠性和生产门禁；
- 阶段完成检查。

## 本文不负责

- 业务功能设计；
- 后端安全实现；
- 具体 Phase 排期。

---

## 1. 测试分层

### 单元测试

覆盖：

- 权限判断；
- Token 过期；
- response normalizer；
- 错误映射；
- 文件大小/类型；
- 状态文案；
- 下载文件名；
- API Readiness。

### 组件测试

覆盖：

- 加载、空、错误和重试；
- 表单校验；
- 401/403/409/422；
- 无证据；
- 引用；
- 轮询停止；
- 未保存离开；
- 危险操作确认。

### 契约测试

- 当前 OpenAPI 与前端类型对照；
- MSW fixture 与 OpenAPI 字段一致；
- raw/envelope 两种响应兼容有测试；
- 后端枚举变化能通过类型或测试失败暴露。

### E2E

```text
1. 匿名深链接 -> 登录 -> 返回
2. sys_admin -> 用户列表 -> 创建 -> 修改角色
3. employee -> 管理页 -> forbidden
4. 提问 -> 引用 -> 反馈
5. 上传文档 -> 索引完成/失败
6. 草稿编辑 -> 确认 -> 导出
7. FAQ 通过/驳回
8. Eval Run -> 轮询 -> 结果
```

4～8 必须等真实 API 就绪；Mock 通过不等于 E2E 完成。

## 2. 安全

- 不用 `dangerouslySetInnerHTML` 直接渲染模型答案；
- Markdown 使用允许列表清洗；
- 外部链接使用 `noopener noreferrer`；
- 上传文件名不作为可信路径；
- 日志不记录 Token、密码、完整敏感正文；
- sessionStorage Token 是过渡方案；
- 403 不能静默；
- 审计和错误详情统一脱敏；
- 只暴露允许的 `VITE_*`；
- 生产关闭 MSW、测试账户和调试数据；
- 前端权限不替代后端鉴权。

## 3. 可靠性

- GET 只有限重试；
- 写操作默认不自动重试；
- 所有写操作防重复；
- 轮询有停止条件；
- 网络错误保留已有缓存；
- 页面卸载取消请求；
- 长任务超时只停止前台等待，不擅自标记后端失败。

## 4. 性能

- 登录页不加载管理模块；
- 路由级懒加载；
- Eval/MCP 独立拆包；
- 列表服务端分页；
- Query 请求去重；
- 聊天列表达到真实性能阈值后再引入虚拟化；
- 没有稳定趋势数据前不引入图表库。

建议验收关注：首屏体积、路由 Chunk、重复请求、长列表和低速网络行为，不为追求单一分数牺牲可用性。

## 5. 可访问性门禁

- 键盘完成核心流程；
- 对话框焦点正确；
- Label 与错误关联；
- 状态不只靠颜色；
- 焦点清晰；
- reduced motion 生效；
- 图标按钮有名称；
- 表格在小屏可访问。

## 6. 每阶段必须通过

- TypeScript；
- lint；
- 单元/组件测试；
- 当前 Phase 的 E2E；
- API 与 OpenAPI 对齐；
- 401/403/422/500 和网络失败有行为；
- 无真实凭据；
- 无未使用大型依赖；
- 无跨 Phase 意外改动。

## 7. F7 生产清单

- 生产 Base URL 正确；
- CORS/反向代理联调；
- Source Map 策略明确；
- 环境变量校验；
- Mock 强制关闭；
- 错误展示 request_id；
- 全量真实 E2E；
- 依赖锁定和漏洞检查；
- 静态资源缓存策略；
- 部署和回滚步骤；
- 未就绪路由关闭；
- 测试数据清除。

## 8. 完成条件

- 每个阶段有可重复的质量结果；
- 不以“页面能打开”代替验收；
- Mock、组件测试和真实 E2E 边界清楚；
- 安全和可访问性不是 F7 才首次检查。
